Project Wild Overview

What This Project Is

Project Wild is a sport analytics project focused on organizing and communicating decision-support work for NHL Wild analysis.

At a high level, it moves through a full analytics lifecycle:

  • define the business context and stakeholder needs,
  • design and prepare data assets,
  • perform modeling and analytical comparison,
  • deliver executive and technical communication artifacts,
  • retain reference material for critique and methodology.

Core Question the Project Addresses

How can structured analytics artifacts support better hockey operations decisions from early problem framing through final strategy communication?

Project Storyline (Start to Finish)

  1. Framing and Discovery

    The project opens with context-setting artifacts that establish stakeholder landscape, business problem framing, requirements, and opportunity discovery.

  2. Data Engineering and Preparation

    The largest body of work is here. Artifacts cover data source landscape, logical modeling, feature engineering, data dictionary/governance, quality assessment, cleaning, and integration into analysis-ready datasets.

  3. Modeling and Analytics

    A focused modeling artifact documents forecasting and comparative analysis workflow.

  4. Communication and Delivery

    The project shifts into value communication, technical handoff, evaluation notes, project summary, and strategy presentation materials intended for executive and operational audiences.

  5. Reference and Supporting Material

    A final reference artifact preserves methodology critique and supporting context.

What the Project Delivers

This project delivers a structured, end-to-end analytics record rather than only a final dashboard or model output.

Primary deliverables include:

  • business and stakeholder alignment artifacts,
  • data architecture and preparation artifacts,
  • modeling walkthrough evidence,
  • executive narrative and strategy communication assets,
  • technical handoff documentation.

How to Read the Project Quickly

If you need a fast understanding path, review in this order:

  1. early framing artifacts (steps 1-5),
  2. data engineering backbone (steps 6-16),
  3. modeling evidence (step 17),
  4. delivery and strategy materials (steps 18-25),
  5. methodology reference (step 26).

Interpretation Notes

  • The strongest project emphasis is on data engineering/preparation and communication/delivery.
  • Modeling evidence is present but comparatively compact in the current artifact set.
  • The collection is suitable for both technical traceability and executive storytelling.
