# Project Wild Case Study

## Project Snapshot

Project Wild is an end-to-end hockey analytics portfolio focused on organizing decision-support work from business framing through strategy communication.

Current evidence base:

- 26 total artifacts
- 16 DOCX, 6 PPTX, 4 XLSX
- Coverage across 5 lifecycle stages

Stage distribution:

- Framing and Discovery: 5
- Data Engineering and Preparation: 11
- Modeling and Analytics: 1
- Communication and Delivery: 8
- Reference and Supporting Material: 1

## The Challenge

Decision-makers need analytics they can trust, not just a final chart or model output.

This project addresses a practical gap: connecting stakeholder questions, data preparation choices, analytical work, and final recommendations into one traceable story.

## The Goal

Build a complete analytics narrative that can support both:

- technical stakeholders who need method and data transparency,
- executive stakeholders who need concise, decision-ready communication.

## Approach

1. Frame the business context
   Artifacts establish stakeholder landscape, requirements, and opportunity framing.
2. Build the data foundation
   The project develops source understanding, data models, governance artifacts, quality checks, cleaning steps, and integrated analysis-ready data.
3. Provide analytical evidence
   A focused modeling artifact captures forecasting/comparative analysis workflow.
4. Translate findings for decisions
   Delivery artifacts convert technical work into value communication, technical handoff, and strategic presentations.

## What the Artifact Set Demonstrates

- Lifecycle continuity from problem framing to final delivery.
- Strong emphasis on data engineering rigor before interpretation.
- Multi-audience communication design (technical and executive).
- Documentation depth that supports auditability and handoff.

## Portfolio Value

As a portfolio case, Project Wild shows more than technical execution.
It shows operating discipline:

- requirements-aware analytics,
- governed data preparation,
- documented methods,
- strategy-ready storytelling.

## Key Takeaway

Project Wild is best understood as a full analytics operating model in artifact form.
Its strongest signal is not a single model output, but the quality of the end-to-end decision support chain.

## Suggested Presentation Flow

When presenting this case study:

1. Open with framing artifacts (steps 1-5) to define the business problem.
2. Highlight the data engineering backbone (steps 6-16).
3. Show modeling evidence (step 17) as the analytical pivot.
4. Close with communication and strategy artifacts (steps 18-25).
5. Reference step 26 as methodology context and critique support.
